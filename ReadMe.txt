' SYSTEM REQUIRMENTS
'\/-\/-\/-\/-\/-\/-\/-\/-\/-\/-\/-\/-\/-\/-\/-\/-\/-\/-\/-\/-\/-\/-\/


  1. At least 64 Mega Bytes of RAM.
  2. At least a 500 MHZ AMD or Intel Pentium 2 or 3 CPU. Or an 
     Intel Celeron of at least 800 MHZ because of the small cache.
  3. Windows 98, 2000, XP, or ME. Windows 95 has not been tested.
  4. Support for 16 bit color.
  5. DirectX 8 or higher. You can get it free at Microsoft.com

' INSTRUCTIONS
'\/-\/-\/-\/-\/-\/-\/-\/-\/-\/-\/-\/-\/-\/-\/-\/-\/-\/-\/-\/-\/-\/-\/


1. If you have WinZip or use Window's built-in Zipping program,
   there may be errors when opening files. This can be anything
   from multiple directories to having all the files in one directory.

2. I suggests using WinRAR if you have this problem.

3. Unzip the Castlevania file onto your desktop.

4. Make sure to set the colors to 16 bit. Right click the desktop,
   click on Properties, then a Display Properties window will pop
   up. Now click on the Settings tab and change the colors to 
   16 bit. Don't use 32 bit, or the animations will be slower.

5. If you have any problems or questions that aren't listed in the 
   known errors section. Please e-mail me at 
   mattjones@workhorsy.org

' KNOWN ERRORS
'\/-\/-\/-\/-\/-\/-\/-\/-\/-\/-\/-\/-\/-\/-\/-\/-\/-\/-\/-\/-\/-\/-\/


1. Sometimes Alucard gets stuck when turning around while attacking 
   or landing.

2. When playing in full screen mode, the program almost always changes 
   the resolution back successfully, but not always.

3. The water only splashes once. This will be fixed when I get the 
   real splash sprite.

4. Zombie explosions hurt Alucard. This is not a glitch. I might even
   make the explosions bigger.

5. If you walk through platforms or stairs that you can jump through, 
   you will sink through them.

6. If you try to go into some rooms, or leave through some passages
   you will crash the game, because there is no file to load, or you'll
   fall off into empty space forever.

7. Some items and candles are in strange places, because I've been too 
   busy fixing engine problems to update them.

8. The Game Over screen is flashing and barely visible, because it is 
    drawn direcly on the screen rather than being first drawn to the
    buffer.

9. The screen drawing is slow and studering due to it using the 
   Windows API and not DirectX.

' CONTROLS
'\/-\/-\/-\/-\/-\/-\/-\/-\/-\/-\/-\/-\/-\/-\/-\/-\/-\/-\/-\/-\/-\/-\/


  X = Attack
  Z = Jump
  Right Arrow = Move Right
  Left Arrow = Move Left
  Down Arrow = Duck
  Esc = Game Over screen
  M = Shows a small hard-to-read menu of items collected
  9 = 0 Delay
  0 = 30 Delay
  1 = 1000 Delay (1 Second)
